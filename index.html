HTML

  • Tags
    • !DOCTYPE HTML: Tells browser this is an HTML page.
    • html: This does not have a closing tag, goes before head and includes:
      • lang="en": This gets included in the html tag.
    • head: Where you include:
      • base: An open tag that displays the base url for the webpage.
      • link: An open tag that links to an external html document such as a stylesheet.
      • meta: Goes inside head tag, doesn't have a closing tag but includes a / before >, and includes elements such as:
        • charset: Character set, the bootcamp course says that UTF-8 is what allows for the most characters and symbols on the page. This sets character encoding, whatever that is.
        • viewport: This sets the default view for the webpage but, I'm not sure I know what that means yet. From what I can tell in the code package for this study guide, it has to do with dimensions.
        • keywords: I don't know what the tag for this is yet but it was mentioned in the video.
        • author: I also don't know what the format of the tag is for this, but it was mentioned that you put author info in this section.
      • style: This is where style tags go for any CSS that is being included on the page itself rather than in a stylesheet.
    • body: This is where everything visible on the site goes. Includes headers, footers, paragaph elements, sections, divs, etc. Whatever will be SEEN.
  • Semantic elements
    • main: Designates the content as the main focus of the page.
    • section: Designates the content as a cohesive section of the page.

CSS Rules


CSS Rules consist of selectors and declarations.


  • property + value = declaration
    • declarations include punctuation of property: selector;
  • selector + declaration = rule

How to use CSS


  • Inline...
    • to style an element directly.
    • which is harder to change because it requires individual handling.
    • by using the style = " " attribute.
    • which has the benefit of being good for quick and permanent changes.
  • Internal CSS stylesheet
    • which is included in the html document itself using the style element tag within the head tags.
    • which allows you to set rules but for that page only!
    • which utilizes CSS classes and IDs to change multiple things on the page using rules.
  • External CSS stylesheet document
    • which is the most common and preferred way to style a webpage.
    • which allows for enveloping changes and keeps design uniform throughout rather than page-by-page.
    • which requires the use of a link element in the head tags to relay the css stylesheet properties.

    How to link an external stylesheet


    Within the head element, add a link element with a rel= attribute that states that it is a stylesheet, and an href= attribute that links the document path.


    Useful tips


    • You can include multiple selectors for a rule by selector 1, -->new lineselector 2
    • * is a wild card selector that applies to every element ever...?
    • DRY code is do not repeat yourself code and this encourages the sharing of pieces of code that need to be written or maintained. Kind of like, reducing a fraction or using object pronouns.
    • Think of html elements like boxes, from outside to inside it's margin, border (like the outline), padding (space between border and content), and content.

Git

  • Use the terminal to do many things concerning git. Always type git before any command, it indicates to the computer where/how to execute the command.
    • Terminal inputs for git
      • git status: Tells you which branch you're on and whether changes have been made and committed.
      • git checkout -b branch-name: Creates a new branch and moves you to that branch.
      • git add -A: This adds all changes you've made to the branch.
      • git commit: This will stick the changes in there for good.
      • git pull origin main: Checks that your current branch is synced with github.
      • git push origin feature/starter-code: Pushes it back to github.
